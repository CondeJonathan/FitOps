# FitOps Frontend Team README

Use this as the single frontend planning and progress document.
Each teammate should update their assigned section as they build.
Do not create per-folder README files; all updates belong in this file.

## Goals

- Keep all frontend planning in one place
- Reduce overlap while working in one shared `main` branch
- Define clear page ownership and expected behavior by role

## Folder Structure

Use this structure when adding files:

- `src/pages/login/`
- `src/pages/member-dashboard/`
- `src/pages/staff-dashboard/`
- `src/pages/schedule/`
- `src/pages/booking/`
- `src/pages/member-status/`
- `src/pages/tickets/`
- `src/pages/maintenance/`
- `src/components/shared/`
- `src/services/`

## Team Update Rules

1. Update this file when you start/finish a feature section.
2. Work in your assigned folder to avoid conflicts.
3. If you must touch a shared file, coordinate with the team first.
4. Keep commits focused on one feature at a time.
